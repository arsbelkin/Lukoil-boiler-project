# Модель бойлера с OPC UA сервером и историей

## Описание
Проект моделирует бойлер с двумя входными потоками (горячая и холодная вода) и одним выходным потоком. Управление осуществляется через регулируемые клапаны на входах и выходе, а также указание температуры на выходе.


## Описание модулей

- **opc** - содержит `opc теги`, `opc сервер` и `opc клиента`

- **model** - содержит `модели клапана и бойлера`, а так же файл для запуска моделирования

- **api** - содержит реализацию `REST-API` с помощью библиотеки `FastAPI`

- **db** - содержит стратегии для работы с БД `SQLlite` и `PostgreSQL`, а так же реализацию `паттерна репозиторий`

- **pid** - содержит реализацию `ПИД-регулятора` для контроля температуры воды посредством изменения процента открытия входных клапанов

- **ui** - содержит `front часть`, написанную на `html, css и js`

## Начальные условия модели

| Параметр | Значение |
| :---: | :---: |
| **Начальный уровень (заполнение)** | `0 %` (бак пуст) |
| **Положение клапана горячей воды** | `50 %` (открыт наполовину) |
| **Положение клапана холодной воды** | `50 %` (открыт наполовину) |
| **Положение выходного (сливного) клапана** | `100 %` (полностью открыт) |
| **Температура горячего источника** | `85 °C` |
| **Температура холодного источника** | `15 °C` |

## Запуск

Для управления виртуальным окружением python используется пакетный менеджер `uv`.


> Возможно придется использовать `python -m pip` или `python -m uv`


1. Установить `uv`
```bash
pip install uv
```

2. Перейти в директорию `src`
```bash
cd src
```

3. Запустить `opc сервер`
```bash
uv run -m opc.run_server
```

4. Запустить `api`
```bash
uv run uvicorn api.api_server:app --reload
```

5. Запустить `модель`
```bash
uv run -m model.run_model
```

6. Запустить `pid-регулятор`
```bash
uv run -m pid.run_pid
```

7. Создать файл `.env` и заполнить его данными по аналогии с `.env.example`

> При использовании PostgreSQL необходимо заранее создать БД

8. Запустить `логгер`
```bash
uv run -m db.history_logger
```
