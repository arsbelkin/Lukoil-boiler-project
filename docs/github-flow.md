# GitHub Flow

**GitHub Flow** - это упрощённая стратегия ветвления в Git, ориентированная на непрерывную поставку (continuous delivery). В отличие от GitFlow, здесь используется всего одна основная ветка и короткоживущие вспомогательные ветки, что делает процесс быстрее и проще для команд с частыми релизами.

## Основная ветка
- `main` : содержит стабильную версию продукта, которая всегда готова к деплою в продакшен.

## Вспомогательные ветки
- `feature/*` (или любое другое описательное имя) : создаются от `main` для разработки новой функции, исправления бага или любых других изменений. После завершения работы и прохождения проверки вливаются обратно в `main`.

Отдельных веток `dev`, `release/*` и `hotfix/*`, как в GitFlow, здесь нет - все изменения, включая срочные исправления, проходят через одну и ту же ветку `main`.

## Рабочий процесс

1. Создается ветка от `main`
```bash
git switch -c feature/*
```

2. Добавляются изменения
```bash
git add .
```

3. Делается коммит
```bash
git commit -m "some changes"
```

4. Получение коммитов из репозитория
```bash
git pull
```

5. Отправка своих коммитов
```bash
git push
```

6. Создается `Pull Request` в ветку `main`

7. После code review и прохождения тестов (CI) ветка сливается (`merge`) в `main`

8. Изменения сразу деплоятся в продакшен
