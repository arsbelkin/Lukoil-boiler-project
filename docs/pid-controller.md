# ПИД-регулятор

**ПИД-регулятор** (PID controller, пропорционально-интегрально-дифференциальный регулятор) - это устройство или алгоритм управления с обратной связью (feedback control), который непрерывно вычисляет **ошибку регулирования** - разницу между заданным значением (уставкой, setpoint) и текущим измеренным значением процесса (например, температурой воды) - и на основе этой ошибки формирует управляющее воздействие (например, угол открытия клапана).

## Основная идея

Регулятор постоянно сравнивает:
- **SP (Setpoint)** - желаемое значение (например, задана температура 40°C)
- **PV (Process Variable)** - текущее измеренное значение (например, датчик показывает 35°C)

Разница между ними называется **ошибкой**:

```
e(t) = SP - PV
```

Задача регулятора - минимизировать эту ошибку, изменяя управляющий сигнал `u(t)` (в вашем случае - степень открытия клапанов горячей/холодной воды).

## Три составляющие

Итоговый управляющий сигнал складывается из трёх слагаемых:

```
u(t) = P + I + D
```

### 1. Пропорциональная составляющая (P)
```
P = Kp * e(t)
```
Реагирует на **текущую** величину ошибки: чем больше отклонение от уставки, тем сильнее воздействие. Обеспечивает быструю реакцию, но сама по себе не может полностью устранить ошибку - обычно остаётся небольшое стационарное отклонение (**static error / offset**).

- Слишком большой `Kp` -> система начинает колебаться и может стать неустойчивой.
- Слишком маленький `Kp` -> регулятор реагирует медленно и вяло.

### 2. Интегральная составляющая (I)
```
I = Ki * ∫e(t)dt
```
Накапливает ошибку **во времени**. Если пропорциональная часть оставляет небольшое постоянное отклонение (например, температура стабильно держится на 39°C вместо заданных 40°C), интегральная составляющая постепенно "дожимает" систему до точного значения, суммируя накопленную ошибку.

- Устраняет статическую ошибку.
- Слишком большой `Ki` может приводить к перерегулированию (overshoot) и медленным затухающим колебаниям - эффект "интегрального насыщения" (integral windup).

### 3. Дифференциальная составляющая (D)
```
D = Kd * de(t)/dt
```
Реагирует на **скорость изменения** ошибки, то есть как быстро приближаемся или удаляемся от уставки. Работает как "тормоз": если температура быстро приближается к заданной, D-составляющая заранее уменьшает воздействие, предотвращая перерегулирование (overshoot) и колебания.

- Делает систему более стабильной и уменьшает колебания.
- Чувствительна к шуму в измерениях (резкие скачки датчика могут вызывать резкие скачки управляющего сигнала), поэтому сигнал перед дифференцированием часто фильтруют.

## Как это работает в применении к бойлеру

1. Датчик измеряет текущую температуру воды на выходе (PV).
2. Регулятор вычисляет ошибку: `e(t) = заданная_температура - текущая_температура`.
3. По формуле ПИД вычисляется управляющий сигнал `u(t)`.
4. Управляющий сигнал преобразуется в изменение степени открытия клапанов горячей и холодной воды (например, увеличить долю открытия горячего клапана, если вода холоднее заданной).
5. Изменение потоков воды влияет на температуру смеси -> новое значение PV -> цикл повторяется с определённым шагом дискретизации.

## Настройка коэффициентов (тюнинг)

Подбор `Kp`, `Ki`, `Kd` определяет качество регулирования:

| Коэффициент | Слишком мал | Слишком велик |
|---|---|---|
| Kp | Медленная реакция, большая статическая ошибка | Колебания, неустойчивость |
| Ki | Долгое устранение статической ошибки | Перерегулирование, медленные колебания |
| Kd | Слабое подавление колебаний, возможен overshoot | Чувствительность к шуму, резкие скачки сигнала |

Типичные методы настройки:
- **Метод Зиглера-Николса** (Ziegler-Nichols) - эмпирический способ подбора коэффициентов на основе реакции системы на предельный (критический) коэффициент усиления.
- **Ручная настройка (trial and error)** - последовательный подбор P, затем I, затем D.
- **Автонастройка (auto-tuning)** - алгоритмы, которые сами подбирают коэффициенты по отклику системы на тестовое воздействие.

## Дискретная форма (для реализации в коде)

На практике регулятор работает с определённым шагом по времени `dt`, поэтому используется дискретная аппроксимация:

```
error = setpoint - process_value
integral += error * dt
derivative = (error - previous_error) / dt
output = Kp*error + Ki*integral + Kd*derivative
previous_error = error
```

Этот псевдокод пересчитывается на каждом шаге (тике) симуляции или каждом цикле опроса датчика.

## Итог

ПИД-регулятор - это способ управления системой, где:
- **P** реагирует на текущую ошибку (скорость реакции),
- **I** устраняет накопленную статическую ошибку (точность в установившемся режиме),
- **D** сглаживает переходный процесс, предсказывая тенденцию изменения ошибки (устойчивость, отсутствие перерегулирования).

Комбинация всех трёх составляющих позволяет добиться быстрого выхода на заданное значение без сильных колебаний и с минимальной итоговой ошибкой.
