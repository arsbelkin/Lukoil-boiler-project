# scada 
1. SCADA — что это в общем?
SCADA (Supervisory Control And Data Acquisition) — это не программа, а комплексная система для:

Сбора данных с датчиков и оборудования (температура, давление, уровень воды).

Визуализации этих данных на экранах оператора (мнемосхемы, графики).

Управления объектами (включить/выключить насос, открыть клапан).

Архивации истории параметров и аварий.

Сигнализации при выходе значений за норму.

Главная задача — дать человеку (оператору) «пульт управления» целым технологическим процессом, часто удаленно.

2. Как работает ВАША SCADA-система (по вашему описанию)
Ваша система — это гибридная SCADA, где каждый компонент отвечает за свой слой. Вот как всё взаимодействует:

🔹 Уровень оборудования (физика)
Бойлер (реальный или симулятор) — источник данных.

С ним общается модель бойлера на Python. Она эмулирует физику процесса (нагрев/остывание) и выставляет параметры через OPC UA-сервер (тоже на Python).

🔹 Уровень связи (стандарт OPC)
OPC UA — это современный промышленный протокол. Ваш Python-сервер делает данные бойлера доступными по этому стандарту.

OPC DA (устаревший) — если он есть, то для совместимости со старыми устройствами, но в вашей схеме, скорее, используется UA.

🔹 Уровень сбора и логики (Сервер)
Сервер на C# — это ядро вашей SCADA. Он:

Подключается к OPC UA-серверу и читает теги (температуру, статус насоса).

Содержит бизнес-логику (правила: если температура > 80°C — отправить команду на отключение).

Пишет команды обратно в OPC UA (управление).

Отдает данные Web-интерфейсу (через REST API или WebSockets).

🔹 Уровень оператора (Визуализация)
Вьюшка (HTML + CSS + JS) — это HMI (Human-Machine Interface). Работает в браузере.

Она запрашивает у C#-сервера актуальные данные и рисует мнемосхему бойлера, графики, индикаторы.

С неё же оператор нажимает кнопки — команда уходит в C#, тот — в OPC, и далее на модель бойлера.

🔹 Уровень хранения (Логирование)
Python-скрипт (или встроенный модуль) подписывается на изменения тегов и пишет историю в SQLite (для локального хранения) и PostgreSQL (для централизованного архива).

Это нужно для отчетов, трендов и анализа аварий.
