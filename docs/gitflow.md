# GitFlow

**GitFlow** - это строгая стратегия ветвление в Git, созданная для управления релизами и командной разработкой. Она разделяет код на две основные ветки (`main` и `dev`) и вспомогательные (`feature/*`, `release/*`, `hotfix/*`), упорядочивая процесс от написания фич до исправления багов.

## Основные ветки
- `main` : содержит стабильную версию продукта, которая находится в продакшене.

- `dev` : ветка интеграции, куда сливаются все новые функции перед релизом.

## Вспомогательные ветки
- `feature/*` : создаются от `dev` для разработки каждой новой функции. После заверешения работы изменения вливаются обратно в `dev`.

- `release/*` : создаются от `dev`, когда функционал для нового релиза готов. Здесь проводится финальное тестирование и правка мелких ошибок. После проверки ветка сливается в `main` и `dev`.

- `hotfix/*` : создаются от `main` для срочного исправления критических ошибок. После испраления багов вливаются в обе основные ветки: `main` и `dev`.


## Рабочий процесс

1. Создается ветка `feature/*`
```bash
git switch -c feature/*
```

2. Добавляеются изменения
```bash
git add .
```

3. Делается коммит
```bash
git commit -m "some changes"
```

4. Получение коммитов из репозитория
```bash
git pull
```

5. Отправка своих коммитов
```bash
git push
```

6. Создается `Pull Request` в ветку `dev`

7. Далее создается `Pull Requests` в ветку `main`
