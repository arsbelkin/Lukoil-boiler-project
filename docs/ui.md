# Документация по интерфейсу (UI)

## Назначение интерфейса

Веб-интерфейс модели бойлера позволяет:
- В реальном времени наблюдать за состоянием бойлера (уровень воды, температура)
- Управлять клапанами (горячая/холодная вода, слив)
- Просматривать историю изменений на графиках

**Целевой пользователь:** оператор / студент / инженер, работающий с моделью через браузер.

---

## Структура страниц

В проекте одна главная страница — **дашборд бойлера**, которая содержит:
1. Визуальную схему бойлера (SVG/Canvas)
2. Панель текущих показателей
3. Панель управления клапанами
4. Графики истории

---

## Компоненты интерфейса

### 1. Визуализация бойлера
- SVG-схема бака с трубами
- Анимированные потоки воды (горячая — красная, холодная — синяя)
- Динамическое заполнение бака в зависимости от `waterLevel`

### 2. Индикаторы
| Параметр | API-поле | Отображение |
|---|---|---|
| Температура горячего входа | `inputHotTemp` | Термометр / число |
| Температура холодного входа | `inputColdTemp` | Термометр / число |
| Выходная температура | `outputTemp` | Термометр / число |
| Уровень воды | `waterLevel` | Прогресс-бар внутри бака |

### 3. Регуляторы клапанов (слайдеры 0–100%)
- `valveHot` — клапан горячей воды
- `valveCold` — клапан холодной воды
- `valveOut` — сливной клапан

При изменении слайдера отправляется `POST /api/v1/data` с `{ name, value }`.

### 4. Графики истории
- **График температуры** — по данным `tempGraph`
- **График уровня воды** — по данным `waterLevelGraph`

Используется библиотека: [Chart.js / Plotly / D3.js — *выбрать*]

## Анимации

- Потоки воды в трубах — CSS-анимация `dash-offset` или SVG `<animate>`
- Плавное изменение уровня воды — `transition: height 0.5s`
- Движение стрелок термометров — `transition: transform` ??

## Адаптивность

| Breakpoint | Поведение |
|---|---|
| `≥ 1024px` | Десктоп: всё в 2–3 колонки |
| `768–1023px` | Планшет: схема сверху, графики снизу |
| `< 768px` | Мобильный: вертикальная прокрутка, упрощённая схема |
